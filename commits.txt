=========================================================
  ARQUITETURA IMPLEMENTADA
=========================================================

Cache Distribuído (Redis - Padrão Cache-Aside):
  GET /medicos  → verifica cache 'medicos:all'  → hit: retorna / miss: DB + popula
  POST /medicos → cria no DB → invalida 'medicos:all'
  GET /pacientes  → verifica cache 'pacientes:all'
  POST /pacientes → cria no DB → invalida 'pacientes:all'

Comunicação em Tempo Real (WebSockets - Socket.IO):
  Namespace: ws://host:3000/notificacoes
  Eventos emitidos pelo servidor:
    agendamento.criado   → { id, medicoId, pacienteId, dataHora, status }
    pagamento.processado → { id, agendamentoId, valor, status, metodoPagamento, dataPagamento }
  Eventos do cliente:
    ping → servidor responde com pong + timestamp
